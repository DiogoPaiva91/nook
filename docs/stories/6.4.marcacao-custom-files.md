# Story 6.4: UX de arquivos "custom" (fora do alcance do builder visual)

## Contexto
Epico 6. Cobre FR18 do PRD (edicoes manuais nao quebram o builder) + AC do epico ("builder reconhece arquivo como custom, nao sobrescreve"). Watcher e a marca ja existem (Story 2.10); aqui formaliza a UX.

## Descricao
Como Diogo, quero que o builder me deixe transitar arquivos entre `managed` e `custom` explicitamente via UI, com indicadores claros, para nao precisar editar `builder.json` na mao.

## Tarefas tecnicas
1. [ ] Aba/painel "Arquivos do projeto" no shell (lista do `files{}` do `builder.json`): coluna path + status (managed/custom/conflict) + acoes.
2. [ ] Acao "marcar como custom": altera `files[path] = { managed:false, reason:"user-marked" }` via patch; substitui header do arquivo por `// @builder:custom`.
3. [ ] Acao "regenerar do builder": apaga arquivo, dispara codegen para re-emitir do schema; alerta se conflito.
4. [ ] Estilizar status com badges: verde managed, amarelo custom, vermelho conflict.
5. [ ] Filtros na lista (por status) + busca por path.

## Arquivos envolvidos
- `public/builder/files-panel.js` - criar (lista + acoes)
- `public/builder/index.html` - modificar (aba files)
- `lib/builder/router.js` - modificar (rotas mark-custom + regenerate)
- `lib/builder/codegen/header.js` - modificar (suporte a `@builder:custom`)

## Definition of Done
- [ ] Lista mostra todos os arquivos do `files{}` com status correto
- [ ] "Marcar custom" altera `builder.json` e header do arquivo
- [ ] "Regenerar" recria arquivo identico ao codegen, perdendo edicoes manuais (com confirmacao)
- [ ] Filtro por status funciona
- [ ] AC do epico: editar arquivo manualmente, builder nao sobrescreve
- [ ] Sem erros no console
- [ ] F5 preserva estado

Complexidade: baixa
