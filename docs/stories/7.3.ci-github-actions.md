# Story 7.3: CI minimo no GitHub Actions (lint + build + testes essenciais)

## Contexto
Epico 7. Cobre uma story esperada do PRD. CI quebra cedo se um PR introduzir regressao — barra protege qualidade do core publico.

## Descricao
Como mantenedor, quero CI no GitHub Actions que roda em cada push/PR (lint + build + testes essenciais) para impedir merges que quebrem o builder.

## Tarefas tecnicas
1. [ ] `.github/workflows/ci.yml` no repo publico: matrix Node 20 + 22, OS Ubuntu. Steps: checkout, setup-node, `npm ci`, lint, build, test.
2. [ ] Adicionar lint: ESLint config minima (`eslint:recommended`); script `npm run lint`. Sem TypeScript no lib do builder (Vanilla JS), so checagens de syntax + bugs comuns.
3. [ ] Snapshot tests do codegen (Story 2.5): rodar codegen sobre `builder.json` fixture e comparar com snapshots commitados. Falha CI se output mudar sem atualizar snapshot.
4. [ ] Smoke test do template: `npm install && npm run build` dentro de `templates/nextjs-supabase/`. Garante que template-base nao apodreceu.
5. [ ] Status badge no README apos primeiro run verde.

## Arquivos envolvidos
- `.github/workflows/ci.yml` - criar
- `.eslintrc.json` - criar
- `package.json` - modificar (scripts lint + test)
- `tests/` - criar (estrutura inicial + snapshots)
- `README.md` - modificar (badge)

## Definition of Done
- [ ] Push em branch dispara o workflow no GitHub Actions
- [ ] Lint, build e tests passam em ambiente limpo
- [ ] PR com codegen mudado sem atualizar snapshot falha
- [ ] Smoke test do template passa
- [ ] Badge "CI: passing" no README
- [ ] Sem erros no console (NA — CI)
- [ ] F5 preserva estado (NA)

## Estimativa
Complexidade: media
