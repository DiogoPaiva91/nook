# Story 3.3: Integrar a tela construida ao Jarvis Hub principal

## Contexto
Epico 3. A tela construida no piloto vive como projeto Next.js separado (`agents-bmad/`). Para fechar o loop, precisa estar acessivel a partir do Hub atual via menu/link, sem reescrever a tela.

## Descricao
Como Diogo, quero acessar a tela de agents BMad direto do menu do Jarvis Hub (mesmo que abra em iframe ou nova aba apontando para o `next dev` do piloto), para usar a feature no fluxo real do dia-a-dia.

## Tarefas tecnicas
1. [ ] Garantir que o `next dev` do projeto-piloto roda em porta fixa reservada (ex.: 4150) via Story 1.5; persistir essa preferencia em `builder_projects.metadata` (campo JSON).
2. [ ] Adicionar item de menu no `public/index.html` "Agents BMad" linkando para `http://localhost:4150/agents` (ou iframe interno em `/hub/agents`).
3. [ ] Auto-start desse preview no boot do Hub (opcional, atras de flag `BUILDER_PILOT_AUTOSTART=1`).
4. [ ] Documentar em `README.md` (raiz do Hub) como rodar o piloto.
5. [ ] Garantir auth do piloto funciona com Supabase local — Diogo loga com email magic link e a sessao persiste.

## Arquivos envolvidos
- `public/index.html` - modificar (item de menu)
- `lib/builder/preview.js` - modificar (suporte a porta fixa por projeto)
- `server.js` - modificar (auto-start opcional)
- `README.md` - modificar (instrucoes piloto)

## Definition of Done
- [ ] Menu do Hub tem item "Agents BMad" funcional
- [ ] Click abre a tela do piloto (em nova aba ou iframe interno)
- [ ] Auto-start no boot funciona se flag setada
- [ ] Login no piloto persiste; refresh nao desloga
- [ ] Hub atual (chat, workers, etc.) continua intacto
- [ ] Sem erros no console
- [ ] F5 preserva estado
