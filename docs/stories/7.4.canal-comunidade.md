# Story 7.4: Canal de comunidade (GitHub Discussions ou Discord)

## Contexto
Epico 7. Cobre G5 do PRD ("comunidade ativa"). PRD deixa decisao simples: Discord vs GitHub Discussions.

## Descricao
Como mantenedor, quero abrir um canal oficial de comunidade para concentrar duvidas, ideias e showcases dos usuarios, separando do tracker de bugs (Issues).

## Tarefas tecnicas
1. [ ] Decidir: **GitHub Discussions** (mais simples, ja no repo, sem moderacao extra) vs **Discord** (mais engajamento, mais trabalho de moderar). Recomendacao Bob: **Discussions** v1, Discord quando >50 contribuidores ativos.
2. [ ] Habilitar Discussions no repo publico; criar 4 categorias: `Announcements`, `Q&A`, `Ideas`, `Show & tell`.
3. [ ] Criar 1 topico fixado em cada categoria com regras + template de issue.
4. [ ] Linkar Discussions no README + CONTRIBUTING + site (Story 7.6).
5. [ ] Adicionar issue templates (.github/ISSUE_TEMPLATE/) para bug-report e feature-request, com link "Para perguntas, use Discussions".

## Arquivos envolvidos
- Configuracao GitHub do repo publico (Discussions ON)
- `.github/ISSUE_TEMPLATE/bug-report.yml` - criar
- `.github/ISSUE_TEMPLATE/feature-request.yml` - criar
- `.github/ISSUE_TEMPLATE/config.yml` - criar (link para Discussions)
- `README.md` - modificar (link Discussions)

## Definition of Done
- [ ] Discussions habilitado e visivel no repo
- [ ] 4 categorias criadas com topico fixado
- [ ] Issue templates aparecem ao abrir nova issue
- [ ] Link "duvidas? va para Discussions" presente no template config
- [ ] README linka Discussions na secao "comunidade"
- [ ] Sem erros no console (NA)
- [ ] F5 preserva estado (NA)

## Estimativa
Complexidade: baixa
