# Story 5.2: Editor de fluxos + palette de passos nativos

## Contexto
Epico 5. Cobre FR16 do PRD (passos nativos). UI estilo n8n/Zapier — canvas com nodes conectados por arestas. Vanilla JS (ADR-4). Vendora `litegraph.js` ou similar standalone se necessario.

## Descricao
Como no-coder, quero arrastar passos para um canvas de workflow, conecta-los com arestas, configurar cada passo via properties panel, para criar fluxos visualmente.

## Tarefas tecnicas
1. [ ] Criar `public/builder/workflows.js`: canvas separado (toggle "Paginas" / "Workflows" no shell). Renderiza nodes da `workflows[i].nodes`, conexoes de `edges`.
2. [ ] Vendora `public/builder/vendor/litegraph.min.js` (ou `drawflow.min.js`) standalone — escolher pelo tamanho/UX. Se decidir DIY, justificar em comentario.
3. [ ] Palette de passos nativos: HTTP request, DB query, DB insert/update/delete, email, storage upload, edge function call, webhook, condition, loop, delay (10 itens v1).
4. [ ] Properties panel reusa Story 2.4: cada step tem schema de campos proprio (URL/method/body para HTTP, table/where para DB, etc.).
5. [ ] Patches em `workflows[].nodes` e `edges` via endpoint existente `/schema/patch`.
