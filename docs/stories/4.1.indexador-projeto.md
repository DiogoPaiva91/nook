# Story 4.1: Indexador de projeto (contexto para Claude)

## Contexto
Epico 4 (Agente). Cobre FR11 do PRD. Pre-requisito de 4.2 (chat com tool use). Componente 7 da arquitetura — alimenta o system prompt com arvore de arquivos + resumo do `builder.json`.

## Descricao
Como Diogo, quero um indexador que monta o contexto do projeto ativo (lista de arquivos relevantes + resumo do `builder.json`) para enviar ao Claude no system prompt, de forma que o agente comece toda interacao ja sabendo o que existe no repo.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/indexer.js` com funcao `buildContext(projectPath)`: lista arquivos respeitando `.gitignore`, devolve `{ tree: [...], builderSummary: { entities, pages, files } }`.
2. [ ] Limites: ignorar `node_modules`, `.next`, `.turbo`, `.git`, `*.log`. Truncar listagem em 500 arquivos com aviso "more files truncated".
3. [ ] Renderizar como markdown no system prompt: arvore (`tree -L 3`) + sumario do `builder.json` (entidades + paginas + files customizados).
4. [ ] Cache: invalidar quando watcher (Story 2.10) reportar mudanca; manter em memoria por projeto ativo.
5. [ ] Endpoint debug `GET /api/builder/projects/:id/context` retornando o markdown gerado.

## Arquivos envolvidos
- `lib/builder/indexer.js` - criar
- `lib/builder/router.js` - modificar (rota debug context)
- `lib/builder/watcher.js` - modificar (emite evento de invalidacao do indexador)

## Definition of Done
- [ ] `GET /api/builder/projects/:id/context` retorna markdown com arvore + sumario
- [ ] Arvore exclui pastas grandes (node_modules etc.) corretamente
- [ ] Editar arquivo invalida cache (proximo `buildContext` re-le)
- [ ] Sumario do `builder.json` lista entidades e paginas com seus paths
- [ ] Tamanho do contexto cabe em 1 mensagem do Claude (< 30k tokens estimados)
- [ ] Sem erros no console
- [ ] F5 preserva estado
