# Story 2.9: Export do projeto como `.zip` standalone

## Contexto
Epico 2. Cobre FR10 do PRD e NFR4 (portabilidade). Exporta o projeto como repo Git pronto para Vercel/Railway/Fly.io/VPS.

## Descricao
Como no-coder, quero baixar o projeto inteiro como `.zip` rodavel fora do Hub, para hospedar em Vercel ou levar para qualquer dev sem dependencias proprietarias.

## Tarefas tecnicas
1. [ ] Endpoint `GET /api/builder/projects/:id/export` em `lib/builder/router.js` que zipa o diretorio do projeto excluindo `node_modules/`, `.next/`, `.turbo/` e devolve com header `Content-Disposition: attachment; filename=<slug>.zip`.
2. [ ] Usar lib `archiver` (nova dep) ou tar + gzip do Node nativo; escolher pela dependencia mais leve.
3. [ ] Garantir inclusao de: `app/`, `lib/`, `db/`, `supabase/migrations/`, `supabase/config.toml`, `middleware.ts`, `package.json`, `package-lock.json`, `.env.example`, `builder.json`, `.mcp.json`, `README.md`, `tsconfig.json`, `next.config.mjs`, `tailwind.config.ts`.
4. [ ] Inicializar `.git` minimo via `simple-git` ou `child_process` antes de zipar (1 commit "initial export from Jarvis Hub builder").
5. [ ] Adicionar botao "Baixar projeto" na UI; estilizar como CTA secundaria.

## Arquivos envolvidos
- `lib/builder/router.js` - modificar (rota export)
- `lib/builder/projects.js` - modificar (helper `exportZip`)
- `public/builder/app.js` - modificar (botao download)
- `package.json` - modificar (dep `archiver` se escolhida)

## Definition of Done
- [ ] `.zip` baixado pesa <50MB para projeto vazio
- [ ] Descompactar + `npm install` + `npm run dev` roda sem nenhum patch
- [ ] Deploy em Vercel via "import Git" funciona apos setar env vars (NFR4)
- [ ] `.env.local` NAO incluido no zip (NFR6a — credenciais nao vazam)
- [ ] `git log` no zip mostra 1 commit inicial
- [ ] Sem erros no console
- [ ] F5 preserva estado
