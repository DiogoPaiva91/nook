# Story 7.6: Pagina publica do projeto (GitHub Pages simples)

## Contexto
Epico 7. Cobre uma story esperada do PRD ("Pagina publica do projeto na fase 1"). Landing simples para receber trafego organico, link de Discussions e CTA "rodar local".

## Descricao
Como visitante curioso, quero uma landing publica do projeto que explique em 30s o que e, mostre 1 GIF, e me direcione para Quickstart e Discussions, para decidir se vale dar `git clone`.

## Tarefas tecnicas
1. [ ] Criar pasta `docs-site/` no repo publico (ou branch `gh-pages`): `index.html` estatico simples + 1 CSS, sem framework.
2. [ ] Conteudo: hero (titulo + tagline + CTA "Rodar local agora"), GIF de 30s do builder em acao, 3 bullets de diferencial (open-source / codigo real / no-code), link Discussions, link GitHub.
3. [ ] Habilitar GitHub Pages (Settings -> Pages -> branch ou `docs-site/`).
4. [ ] Adicionar dominio custom opcional (`builder.jarvis.dev` se Diogo quiser, senao `<user>.github.io/<repo>`).
5. [ ] Meta tags OG/Twitter para preview ao compartilhar links.

## Arquivos envolvidos

- `docs-site/style.css` - criar
- `docs-site/og-image.png` - criar (preview social)
