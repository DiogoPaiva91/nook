# Story 4.6: Rate limit + telemetria basica de tokens

## Contexto
Epico 4. Cobre AC do epico (uso saudavel) e prepara base para NFR7 da fase 2 (custo por usuario). Plano Max do Diogo nao tem limite explicito, mas telemetria evita queima descuidada.

## Descricao
Como Diogo, quero registrar tokens consumidos por mensagem/projeto/dia e limitar requests concorrentes, para acompanhar custo e nao queimar a conta com loops acidentais.

## Tarefas tecnicas
1. [ ] Adicionar tabela `builder_agent_usage (id, project_id, message_id, input_tokens, output_tokens, cost_estimate_cents, created_at)` em `lib/builder/db.js`.
2. [ ] Capturar usage do `claude` CLI via stream-json (ja vem no output) em `lib/builder/agent.js`; persistir apos cada mensagem.
3. [ ] Rate limit: max 1 mensagem em voo por projeto; max 60 mensagens/hora globalmente (config em `data/config.json` ou env). Bloquear nova mensagem se limite atingido.
4. [ ] Endpoint `GET /api/builder/projects/:id/agent/usage` para consumo do projeto + global; UI exibe em rodape do chat.
5. [ ] Logar warning no console quando 80% do limite atingido.

## Arquivos envolvidos
- `lib/builder/db.js` - modificar (tabela usage)
- `lib/builder/agent.js` - modificar (capturar usage + rate limit)
- `lib/builder/router.js` - modificar (rota usage)
- `public/builder/chat.js` - modificar (rodape com tokens)

## Definition of Done
- [ ] Cada mensagem registra input/output tokens corretos em `builder_agent_usage`
- [ ] Mandar 2 mensagens em paralelo no mesmo projeto bloqueia a 2a com erro 429
- [ ] Rodape do chat mostra tokens hoje + estimativa USD
- [ ] Warning no console aos 80%
- [ ] Reset diario do contador (cron simples ou comparacao por data ao ler)
- [ ] Sem erros no console
- [ ] F5 preserva estado
