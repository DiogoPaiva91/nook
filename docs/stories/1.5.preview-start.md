# Story 1.5: Endpoint de start do preview (`next dev` em subprocess)

## Contexto
Story do Epico 1. Cobre Componente 5 (Preview Manager) da arquitetura. ADR-5: subprocess isolado por projeto, porta 4100-4199, iframe na UI. Inclui spike de cache de `node_modules` (item 2 da §10 da arquitetura — 2h).

## Descricao
Como no-coder, quero clicar "Preview" e ver o app rodando numa URL local em ate 30s na primeira vez, para validar visualmente o que o builder gerou.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/preview.js` com pool de processos: `start(projectId)` aloca porta livre no range 4100-4199, registra em `builder_preview_processes`, spawna `npm install` (so se `node_modules` ausente) seguido de `npm run dev -- -p <porta>` no `cwd` do projeto.
2. [ ] Spike (timeboxed 2h): testar symlink global de `node_modules` compartilhado vs `npm install` por projeto; documentar decisao em comentario no topo de `preview.js`.
3. [ ] Endpoint `POST /api/builder/projects/:id/preview/start` registrado no router; retorna `{ url: "http://localhost:<porta>", status: "starting" | "ready" }` apos health check (ping na porta com retries).
4. [ ] Stream de logs do `next dev` para SSE em `/api/builder/projects/:id/preview/logs` (reusa Worker Bus se simples; senao implementacao direta).
5. [ ] UI: botao "Preview" em `public/builder/app.js` chama o endpoint, mostra spinner "compilando…" ate `ready`, depois abre iframe.

## Arquivos envolvidos
- `lib/builder/preview.js` - criar (pool + port allocator + spawn)
- `lib/builder/router.js` - modificar (rotas preview start + logs)
- `public/builder/app.js` - modificar (botao preview + iframe)
- `public/builder/style.css` - modificar (estilos do iframe + spinner)

## Definition of Done
- [ ] Primeiro start (com `npm install`) sobe preview em <= 30s (NFR/AC do epico)
- [ ] Subsequentes starts sobem em <= 5s (skip install)
- [ ] Acessar a URL retornada renderiza a home do template
- [ ] `SELECT * FROM builder_preview_processes` mostra a entrada com pid + porta corretos
- [ ] Tentar start enquanto ja rodando reusa a porta existente (idempotente)
- [ ] Sem erros no console
- [ ] F5 preserva estado (preview continua rodando)
