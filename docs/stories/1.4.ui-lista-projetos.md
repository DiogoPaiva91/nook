# Story 1.4: UI inicial "Meus Projetos" do builder

## Contexto
Story do Epico 1. Cobre o shell visual em `public/builder/` (Componente 8 ainda no esqueleto). Vanilla JS, sem bundler (ADR-4). Consome os endpoints `GET /api/builder/projects` e `POST /api/builder/projects` da Story 1.2.

## Descricao
Como Diogo, quero abrir `http://localhost:<porta>/builder/` e ver a lista de projetos criados, com botao "Novo Projeto", para entrar no fluxo do Modo de Codigo sem usar a UI principal do Hub.

## Tarefas tecnicas
1. [ ] Criar `public/builder/index.html` com shell minimo (header, container central) seguindo o padrao visual do `public/index.html` atual.
2. [ ] Criar `public/builder/app.js` com bootstrap: fetch `GET /api/builder/projects`, renderizar lista (cards com nome + path + data).
3. [ ] Implementar modal "Novo Projeto" com inputs `name` e `parentDir` (default = `~/dev/builder-projects/`); submit chama `POST /api/builder/projects` e re-renderiza a lista.
4. [ ] Adicionar rota `GET /builder/` no `lib/builder/router.js` que serve `public/builder/index.html`; rota `GET /builder/<file>` serve estaticos da pasta.
5. [ ] Linkar a partir do `public/index.html` (Hub atual) um botao/atalho discreto para abrir `/builder/` em nova aba.

## Arquivos envolvidos
- `public/builder/index.html` - criar (shell)
- `public/builder/app.js` - criar (bootstrap + lista + modal)
- `public/builder/style.css` - criar (estilos minimos do shell)
- `lib/builder/router.js` - modificar (servir estaticos `/builder/*`)
- `public/index.html` - modificar (atalho para `/builder/`)

## Definition of Done
- [ ] Abrir `/builder/` mostra lista (mesmo que vazia) sem erro JS
- [ ] Criar projeto via modal aparece imediatamente na lista apos sucesso
- [ ] Recarregar a pagina mantem a lista (vem do SQLite, nao memoria)
- [ ] Atalho do Hub abre nova aba em `/builder/`
- [ ] WCAG AA basico: tab navega pelo modal, esc fecha
- [ ] Sem erros no console
- [ ] F5 preserva estado
