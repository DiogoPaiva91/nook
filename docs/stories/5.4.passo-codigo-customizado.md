# Story 5.4: Passo "codigo customizado" gerado pelo agente

## Contexto
Epico 5. Cobre FR16 do PRD (passo customizado via agente). Depende do Epico 4 (Agente). Permite que o usuario escreva o que quer em linguagem natural e o agente gere o trecho que entra no fluxo.

## Descricao
Como no-coder, quero adicionar um step "codigo customizado" cujo corpo eu descrevo em linguagem natural e o agente gera, para resolver casos que a palette nativa nao cobre.

## Tarefas tecnicas
1. [ ] Adicionar tipo `custom_code` ao schema (Story 5.1) com props `{ description: string, generatedCode?: string, language: 'ts'|'sql' }`.
2. [ ] No properties panel, oferecer textarea "Descreva o que esse passo deve fazer" + botao "Gerar codigo (agente)".
3. [ ] Botao chama endpoint novo `POST /api/builder/projects/:id/agent/generate-step` que envia ao Claude descricao + contexto do workflow; retorna `generatedCode` (string).
4. [ ] Codegen (Story 5.3) emite o `generatedCode` literalmente como bloco no codigo, marcado com comentario `// @builder:custom-step <id>` e arquivo associado entra como `managed:false` para nao ser regenerado se o usuario editar.
5. [ ] UI mostra preview do codigo gerado e pede confirmacao antes de salvar no schema.

## Arquivos envolvidos
- `lib/builder/schema.js` - modificar (custom_code step)
- `lib/builder/agent.js` - modificar (helper `generateStep`)
- `lib/builder/router.js` - modificar (rota generate-step)
- `lib/builder/codegen/workflow.js` - modificar (emitir custom_code)
- `public/builder/properties.js` - modificar (UI custom_code)

## Definition of Done
- [ ] Step custom_code com descricao "calcule hash sha256 do input" gera codigo TS funcional
- [ ] Codigo gerado aparece em preview antes de salvar
- [ ] Apos salvar, step e codegen emitem o trecho no fluxo
- [ ] Editar codigo gerado mantem step com `generatedCode` atualizado (sem regenerar via agente)
- [ ] AC do epico: workflow com custom_code passa `npm run build`
- [ ] Sem erros no console
- [ ] F5 preserva estado
