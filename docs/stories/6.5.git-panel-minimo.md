# Story 6.5: Git panel minimo (status + commit + push)

## Contexto
Epico 6. Cobre uma story esperada (`6.5 — Git panel minimo`). Sem Git integrado, o usuario precisa abrir terminal — quebra a promessa de "tudo no builder".

## Descricao
Como Diogo, quero um painel Git basico no builder (status, stage, commit, push) para versionar o projeto sem sair do Hub.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/git.js` wrappando `simple-git` (npm) ou comandos `git` via `child_process`. Funcoes: `status`, `add`, `commit`, `push`, `branch`, `log` (ultimas 10).
2. [ ] Endpoints `GET /api/builder/projects/:id/git/status`, `POST .../git/commit { message, files[] }`, `POST .../git/push`, `GET .../git/log`.
3. [ ] UI `public/builder/git.js` (toggle de painel): listar arquivos modificados/staged, checkbox para selecionar, textarea de mensagem, botao Commit + botao Push.
4. [ ] Inicializar repo automaticamente no primeiro commit se nao existir (`.git` ausente). Configurar autor com env `GIT_AUTHOR_NAME`/`GIT_AUTHOR_EMAIL` ou prompt na primeira vez.
5. [ ] Suporte a remote: campo "git remote add origin <url>" no painel; persistir como `metadata.gitRemote` em `builder_projects`.

## Arquivos envolvidos
- `lib/builder/git.js` - criar
- `lib/builder/router.js` - modificar (rotas git)
- `public/builder/git.js` - criar (UI panel)
- `public/builder/index.html` - modificar (toggle git)
- `package.json` - modificar (`simple-git` se escolhido)

## Definition of Done
- [ ] Listar status mostra arquivos modificados/untracked corretamente
- [ ] Selecionar + commit gera commit no `git log`
- [ ] Push para remote configurado funciona (ou erro claro se sem remote)
- [ ] Configurar remote pela UI persiste e habilita push
- [ ] Init automatico cria `.git` no primeiro commit se ausente
- [ ] Sem erros no console
- [ ] F5 preserva estado

Complexidade: media
