# Story 2.1: Schema canonico do `builder.json` + lock concorrente

## Contexto
Pivot tecnico do Epico 2 (Builder MVP). Define a representacao intermediaria que vira source of truth (ADR-2). Resolve item 1 da §10 da arquitetura (lock concorrente). Depende de 1.3 ja ter `builder.json` vazio no template.

## Descricao
Como mantenedor do builder, quero um schema documentado e validado de `builder.json` + utilitarios de leitura/escrita atomicos com lock, para que codegen e patches operem sem corromper o arquivo em concorrencia.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/schema.js` com JSON Schema (draft-07) do `builder.json` v1: `version`, `project`, `entities[]`, `pages[]`, `files{}`. Refinar a partir do rascunho da §6 da arquitetura.
2. [ ] Adicionar validador (`ajv` ou validador manual leve) e exportar `validate(builderJson): { ok, errors }`.
3. [ ] Criar `lib/builder/store.js` com `read(projectPath)` / `writeAtomic(projectPath, json)` usando `proper-lockfile` (npm) ou advisory lock SQLite. Decisao documentada em comentario no topo.
4. [ ] Adicionar API de patch (RFC 6902 JSON Patch) `applyPatch(projectId, patches[])`: le, valida, aplica, valida resultado, escreve atomico, retorna novo schema + sha.
5. [ ] Endpoint `POST /api/builder/projects/:id/schema/patch` aceitando array de operacoes JSON Patch.

## Arquivos envolvidos
- `lib/builder/schema.js` - criar (JSON Schema + validacao)
- `lib/builder/store.js` - criar (read/write atomico + lock)
- `lib/builder/router.js` - modificar (rota schema/patch)
- `package.json` - modificar (adicionar `proper-lockfile` + `ajv` se escolhidos)

## Definition of Done
- [ ] `validate({})` rejeita; `validate(template-vazio)` aceita
- [ ] Patch valido aplica e persiste; patch invalido devolve 400 sem alterar arquivo
- [ ] Duas escritas concorrentes nao corrompem (teste manual com 2 curls em paralelo)
- [ ] sha do schema retornado bate com `sha256(JSON.stringify(builderJson)).slice(0,8)`
- [ ] Documento `docs/architecture.md` §6 atualizado (link ou nota apontando schema canonico)
- [ ] Sem erros no console
- [ ] F5 preserva estado
