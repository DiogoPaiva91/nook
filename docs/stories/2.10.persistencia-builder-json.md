# Story 2.10: Persistencia + watcher reconciliando `builder.json`

## Contexto
Epico 2. Cobre Componente 6 (File Watcher + Sync) da arquitetura. Mitiga o risco "watcher entra em loop" listado na §8 da arquitetura. Story tecnica que costura 2.5 (codegen) com a possibilidade de o usuario/agente editar arquivos manualmente.

## Descricao
Como mantenedor do builder, quero um watcher que detecta edicoes manuais em arquivos `@builder:managed` e marca como `custom` em `builder.json` para o builder nunca sobrescrever sem confirmacao do usuario.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/watcher.js` com `start(projectPath)` usando `fs.watch` recursivo; debounce de 200ms por arquivo.
2. [ ] Para cada change: ler arquivo, extrair header `@builder:managed schema=<sha>`. Calcular sha esperado a partir do `builder.json` atual; se diferente -> conflito.
3. [ ] Em conflito, atualizar `files{}` em `builder.json`: `{ managed: false, reason: "manual-edit" }`. UI dispara prompt: "manter custom" (default) ou "regenerar do builder (sobrescreve)".
4. [ ] Garantir que codegen registra hash esperado **antes** de escrever, e watcher ignora writes proprios comparando hash (mitigacao do loop).
5. [ ] Endpoint `GET /api/builder/projects/:id/files/status` lista arquivos com status `managed | custom | conflict`. UI exibe indicador na barra superior.

## Arquivos envolvidos
- `lib/builder/watcher.js` - criar (fs.watch + reconcile)
- `lib/builder/codegen/index.js` - modificar (registrar hash antes de escrever)
- `lib/builder/router.js` - modificar (rota status)
- `public/builder/app.js` - modificar (UI de status + prompt de conflito)

## Definition of Done
- [ ] Editar `app/home/page.tsx` manualmente marca arquivo como `custom` em `builder.json`
- [ ] Builder nao sobrescreve arquivo `custom` em patches subsequentes
- [ ] Confirmar "regenerar" forca reescrita e volta o arquivo a `managed`
- [ ] Codegen interno NAO dispara loop no watcher (verificavel rodando 100 patches em 10s)
- [ ] UI mostra badge "custom" e "conflict" para arquivos respectivos
- [ ] Sem erros no console
- [ ] F5 preserva estado
