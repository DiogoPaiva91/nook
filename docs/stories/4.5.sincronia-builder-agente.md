# Story 4.5: Sincronia builder <-> agente (FR14)

## Contexto
Epico 4. Cobre FR14 do PRD: edicoes do agente atualizam o `builder.json` se representaveis, senao marcam o arquivo como `custom`. Componente 6 + 7 cooperando. Risco "edicao parcial vira custom em excesso" (§8 da arquitetura) e mitigado por componentes pequenos por arquivo (Story 2.5).

## Descricao
Como Diogo, quero que apos aceitar um diff do agente, o builder atualize automaticamente o `builder.json` se a edicao for representavel no schema, OU marque o arquivo como `custom` se nao for, sem que builder e agente "briguem" pelo mesmo arquivo.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/sync.js` com `reconcile(projectPath, filePath, newContent)`: tenta fazer parse reverso simples (regex/heuristica) para detectar edicoes representaveis (ex.: trocar `text` de Heading) e gerar JSON Patch correspondente.
2. [ ] Se reconcile encontra patch, aplica em `builder.json` via `store.applyPatch` e regenera o arquivo (codegen volta a controlar). Se nao, marca `files[path] = { managed:false, reason:"agent-custom" }`.
3. [ ] Hook em `agent.js`: ao aceitar mudanca via Story 4.4, chamar `reconcile` antes do write final.
4. [ ] Watcher (Story 2.10) tem precedencia sobre sync para detectar conflitos com hash desatualizado — arquivo ja em conflito nao tenta reconcile.
5. [ ] UI mostra apos accept: "1 arquivo sincronizado com builder, 2 arquivos custom" para feedback claro do usuario.

## Arquivos envolvidos
- `lib/builder/sync.js` - criar (reconcile)
- `lib/builder/agent.js` - modificar (chamar reconcile no accept)
- `lib/builder/diff.js` - referenciado
