# Story 5.5: Testador de workflow com inputs mockados

## Contexto
Epico 5. Cobre uma story esperada do PRD (testador). Sem testador, o usuario so descobre erros depois do deploy. Reusa o `next dev` do preview (Story 1.5).

## Descricao
Como no-coder, quero rodar um workflow com inputs de teste a partir do builder e ver os outputs/erros de cada step, para depurar visualmente sem abrir terminal.

## Tarefas tecnicas
1. [ ] UI: botao "Testar" em cada workflow no canvas; modal com form de inputs mockados conforme tipo de trigger (HTTP -> body/headers; DB -> registro mock; cron -> instant).
2. [ ] Endpoint `POST /api/builder/projects/:id/workflows/:wid/test` que executa o workflow no contexto do `next dev` ativo: para HTTP triggers, faz request real; para DB/cron, dispara endpoint sintetico exposto em dev mode.
3. [ ] Codegen (Story 5.3) gera, em modo dev, endpoints `/api/_dev/test-workflow/<slug>` que executam o workflow com payload do body (so disponivel se `NODE_ENV !== 'production'`).
4. [ ] Coletar logs por step (timing, output, erros) e enviar via SSE para a UI; renderizar como timeline lateral.
5. [ ] Indicador visual no canvas: nodes verdes (sucesso), vermelhos (erro), cinzas (nao executado).

## Arquivos envolvidos
- `public/builder/workflows.js` - modificar (botao testar + timeline)
- `lib/builder/router.js` - modificar (rota test workflow)
- `lib/builder/codegen/workflow.js` - modificar (endpoint sintetico em dev)
- `public/builder/style.css` - modificar (cores nodes)

## Definition of Done
- [ ] Testar workflow simples (HTTP -> log) retorna output correto
- [ ] Erro em step vermelho com mensagem visivel
- [ ] Timeline mostra tempo de cada step
- [ ] Endpoint `/api/_dev/test-workflow/*` nao existe em build de producao
- [ ] Inputs mockados validados conforme tipo do trigger
- [ ] Sem erros no console
- [ ] F5 preserva estado
