# Story 1.2: Endpoint de criacao de projeto do builder

## Contexto
Story do Epico 1. Depende de 1.1 (router + metadata) e 1.3 (template-base). Materializa FR1 do PRD: criar projeto Next.js + Supabase a partir do Modo de Codigo, registrando em metadata. Componente afetado: 2 (Metadata) + uso do 3 (Template).

## Descricao
Como no-coder, quero um endpoint `POST /api/builder/projects` que copia o template-base para uma pasta escolhida e registra o projeto, para conseguir criar um app web do zero sem CLI.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/projects.js` com funcao `createProject({ name, parentDir })`: gera id (uuid v4), define path = `<parentDir>/<slug-do-name>`, valida que pasta nao existe.
2. [ ] Implementar copia recursiva de `templates/nextjs-supabase/` para o path do projeto + interpolacao simples (substituir `{{PROJECT_NAME}}` em `package.json`, `README.md`, `builder.json`).
3. [ ] Inserir registro em `builder_projects` (id, name, path, stack='nextjs-supabase', created_at, updated_at).
4. [ ] Registrar rota `POST /api/builder/projects` no `lib/builder/router.js` chamando `projects.createProject` e devolvendo 201 com o registro JSON. Tratar erros (pasta existe, name invalido, sem permissao).
5. [ ] Endpoint complementar `GET /api/builder/projects` listando todos os projetos do SQLite.

## Arquivos envolvidos
- `lib/builder/projects.js` - criar (CRUD inicial)
- `lib/builder/router.js` - modificar (registrar 2 rotas)
- `templates/nextjs-supabase/` - referenciado (criado em 1.3)

## Definition of Done
- [ ] `POST /api/builder/projects {name:"piloto"}` cria pasta no disco e registro no SQLite
- [ ] Tentar criar com nome duplicado retorna 409
- [ ] `GET /api/builder/projects` lista o projeto recem-criado
- [ ] Estrutura de arquivos copiada bate com o template (ver tree)
- [ ] NFR6 respeitado: nenhuma escrita fora de `parentDir`
- [ ] Sem erros no console
- [ ] F5 preserva estado
