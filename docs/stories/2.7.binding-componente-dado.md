# Story 2.7: Binding componente <-> dado (Lista le entidade, Form cria/edita)

## Contexto
Epico 2. Cobre FR8 do PRD. Conecta Story 2.4 (properties panel) + 2.6 (entidades) + 2.5 (codegen). Cobre tambem FR10c (realtime opcional em listas).

## Descricao
Como no-coder, quero conectar uma Lista a uma entidade e um Form a operacao create/update sobre essa entidade, com opcao de realtime, para ter CRUD funcionando sem escrever codigo.

## Tarefas tecnicas
1. [ ] No properties panel, expor campo `entity` (select com entidades disponiveis) para `List` e `Table`; campo `entity` + `mode: create|edit` + `redirectTo` para `Form`.
2. [ ] Para `List`: incluir checkbox "Atualizar em tempo real" (FR10c) que adiciona `realtime: true` nas props.
3. [ ] Estender `lib/builder/codegen/component.js`: List conectada -> componente cliente que faz `select * from <table>` via Supabase client + map para `<li>`. Form conectado -> server action que insere/atualiza usando Drizzle + redirect.
4. [ ] Criar `lib/builder/codegen/realtime.js` que adiciona `supabase.channel(...).on('postgres_changes', ...)` em listas marcadas.
5. [ ] Validacao: se entidade referenciada nao existir mais em `entities[]`, codegen rejeita o patch e UI mostra aviso.

## Arquivos envolvidos
- `public/builder/properties.js` - modificar (campos entity, mode, realtime)
- `lib/builder/codegen/component.js` - modificar (List/Table/Form com binding)
- `lib/builder/codegen/realtime.js` - criar
- `lib/builder/codegen/index.js` - modificar (incluir realtime)

## Definition of Done
- [ ] Lista conectada a `Agent` renderiza registros do Supabase em preview
- [ ] Form conectado em modo `create` insere registro e redireciona
- [ ] Marcar "tempo real" e inserir registro via SQL faz lista atualizar sem reload
- [ ] Codigo gerado segue patterns idiomaticos (`@supabase/ssr` para server, browser client para realtime)
- [ ] Quebra de referencia (entidade deletada) mostra erro na UI antes de gerar codigo invalido
- [ ] Sem erros no console
- [ ] F5 preserva estado
