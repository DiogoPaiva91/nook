# Story 6.1: Integrar Monaco Editor (standalone) no builder

## Contexto
Epico 6 (Editor completo). Cobre FR17 do PRD parcialmente (editor). ADR-4 + §9 da arquitetura: Monaco vendorado standalone em `public/builder/vendor/monaco-editor/`. Sem bundler.

## Descricao
Como Diogo, quero abrir qualquer arquivo do projeto num editor Monaco dentro do builder, com syntax highlight basico, para inspecionar/editar codigo quando o canvas nao basta.

## Tarefas tecnicas
1. [ ] Baixar Monaco Editor standalone (versao AMD ou ESM-loader) e vendorar em `public/builder/vendor/monaco-editor/`. Versao pinada.
2. [ ] Criar `public/builder/editor.js`: abre Monaco em painel (modal full-screen ou tab adicional). Recebe path -> faz `GET /api/builder/projects/:id/file?path=...` -> exibe.
3. [ ] Suporte a escrita: salvar (Ctrl+S) chama `PUT /api/builder/projects/:id/file?path=...` com novo conteudo. Watcher (Story 2.10) detecta e marca `custom`.
4. [ ] Adicionar abas (multiplos arquivos abertos) com indicador de "modificado nao salvo". Fechar aba pede confirmacao se modificado.
5. [ ] UI: clicar em arquivo no painel de status (Story 2.10) abre no Monaco; tambem botao "ver codigo" no properties panel abre o `.tsx` da pagina ativa.

## Arquivos envolvidos
- `public/builder/vendor/monaco-editor/` - criar (vendor)
- `public/builder/editor.js` - criar
- `lib/builder/router.js` - modificar (rotas file get/put)
- `public/builder/index.html` - modificar (container do editor)
- `public/builder/style.css` - modificar (layout editor)

## Definition of Done
- [ ] Abrir `app/home/page.tsx` no editor mostra conteudo com syntax highlight
- [ ] Editar + Ctrl+S salva no disco
- [ ] Watcher reage e marca arquivo `custom` em `builder.json`
- [ ] Multi-aba funcional, fechar com modificado pendente confirma
- [ ] "Ver codigo" do properties panel abre arquivo correto
- [ ] Sem erros no console
- [ ] F5 preserva estado (abas reabrem)
