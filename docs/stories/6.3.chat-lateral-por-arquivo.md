# Story 6.3: Chat lateral por arquivo (reusa Epico 4)

## Contexto
Epico 6. Cobre FR17 ("chat lateral com contexto do arquivo aberto"). Reusa Story 4.2 (endpoint chat) e 4.3 (UI chat) — adiciona contexto especifico do arquivo aberto no Monaco.

## Descricao
Como Diogo, quero ter um chat lateral no editor Monaco que entende qual arquivo esta aberto e o que esta selecionado, para pedir refatorings com contexto preciso.

## Tarefas tecnicas
1. [ ] Estender `lib/builder/agent.js` com parametro opcional `fileContext: { path, content, selection: { startLine, endLine, text } }`. Quando presente, system prompt inclui o snippet selecionado em destaque.
2. [ ] UI no editor: painel lateral (toggle) que envia `selection` atual junto com a mensagem. Reusa componente de chat da Story 4.3.
3. [ ] Botao quick-action "Explicar selecao" / "Refatorar para X" / "Adicionar tipos" — atalhos que pre-preenchem prompt comum.
4. [ ] Diff resultante (Story 4.4) prioriza mostrar o arquivo aberto primeiro e re-foca apos accept.
5. [ ] Historico do chat fica vinculado ao projeto, mas mensagens com `fileContext` ganham label do arquivo no historico.

## Arquivos envolvidos
- `lib/builder/agent.js` - modificar (param fileContext)
- `public/builder/editor.js` - modificar (painel lateral chat)
- `public/builder/chat.js` - referenciado (componente reusado)

## Definition of Done
- [ ] Selecionar 5 linhas e mandar "explicar" envia o snippet ao agente
- [ ] Resposta menciona corretamente o codigo selecionado (sem alucinar)
- [ ] Quick-actions geram prompts pre-preenchidos
- [ ] Diff aplicado refaz mark de `custom` ou re-sincroniza (Story 4.5)
- [ ] Historico mostra label do arquivo na mensagem
- [ ] Sem erros no console
