# Story 3.2: Construir a tela de gerenciamento de agents BMad usando o builder

## Contexto
Epico 3 (Piloto / gate de validacao da fase 1). Cobre FR19 do PRD. Esta story **nao escreve codigo do builder** — usa o builder pronto (Epico 1+2) para construir a feature. Sucesso aqui = validacao da G6.

## Descricao
Como Diogo no papel de usuario do builder, quero construir a tela de listar/abrir/editar/criar/deletar agents BMad arrastando componentes e definindo a entidade `Agent`, sem editar nenhum arquivo `.tsx`/`.ts`/`.sql` manualmente.

## Tarefas tecnicas
1. [ ] No builder, criar projeto novo "agents-bmad" usando o template-base (Story 1.2).
2. [ ] Definir entidade `Agent` no modelo de dados visual (Story 2.6) com campos coerentes com 3.1 (id, name, prompt, file_path).
3. [ ] Montar pagina `/agents`: Container -> Heading "Agentes" + List (entity=Agent, realtime=false). Cada item da lista navega para `/agents/[id]` (modo edicao).
4. [ ] Montar pagina `/agents/[id]`: Form (entity=Agent, mode=edit) + Button "Excluir" (com confirm dialog). Pagina `/agents/new`: Form (mode=create).
5. [ ] Configurar auth: pagina protegida (auth=required) para `/agents` e filhos (FR10a).

## Arquivos envolvidos
- Projeto-piloto `agents-bmad/` (todo gerado pelo builder, nao editado manualmente)
- `bmad/agents/*.md` - fonte (via adapter da 3.1)

## Definition of Done
- [ ] Listar agents funciona em preview (le do BMad real ou da tabela synced)
- [ ] Criar agent persiste corretamente
- [ ] Editar prompt e salvar atualiza fonte
- [ ] Excluir com confirmacao remove o agent
- [ ] **Zero arquivos `.tsx`, `.ts`, `.sql`, `.prisma` editados manualmente** (verificavel via `git log` do projeto-piloto + headers `@builder:managed`)
- [ ] Tela acessivel apenas com login (auth=required funcionando)
- [ ] Sem erros no console
- [ ] F5 preserva estado
