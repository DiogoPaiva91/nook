# Story 7.5: Documentacao "Quickstart" (instalar + primeiro projeto em <= 10min)

## Contexto
Epico 7. Cobre AC do epico: "primeiro projeto criado em <= 10 min por usuario novo". Onboarding e o que define se o repo publico tem tracao ou morre depois do star.

## Descricao
Como usuario novo, quero seguir um guia de 10 minutos para instalar o builder + criar meu primeiro app + ver rodando, para sentir valor antes de desistir.

## Tarefas tecnicas
1. [ ] `docs/quickstart.md` no repo publico: 5 secoes — Instalacao (3 comandos), Criar projeto, Configurar Supabase (escolha local-CLI ou Cloud), Montar 1 tela simples, Ver preview.
2. [ ] Cada secao com screenshots/GIFs curtos. Capturar com `peek` ou similar.
3. [ ] Solucionar problemas comuns no final (Node version, porta ocupada, Supabase CLI nao instalado).
4. [ ] Tempo cronometrado (Bob ou Mary fazendo dry-run em maquina limpa) registrado em `docs/quickstart-time.md`.
5. [ ] Linkar do README + Discussions (anuncio inicial).

## Arquivos envolvidos
- `docs/quickstart.md` - criar
- `docs/assets/quickstart-*.png|gif` - criar (screenshots)
- `docs/quickstart-time.md` - criar (medicao do dry-run)
- `README.md` - modificar (linkar quickstart)

## Definition of Done
- [ ] Quickstart completo segue do zero a `npm start` + 1 tela em preview
- [ ] Dry-run em maquina limpa fica em <= 10min (FR21 + AC do epico)
- [ ] Screenshots/GIFs claros em cada secao
- [ ] Troubleshooting cobre 3 erros mais comuns (testar quebrando proposital)
- [ ] Tempo do dry-run documentado
- [ ] Sem erros no console (NA)
- [ ] F5 preserva estado (NA)
