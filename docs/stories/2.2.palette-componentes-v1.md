# Story 2.2: Palette com componentes v1 (Container, Texto, Botao, Input, Form, Lista, Tabela, Imagem)

## Contexto
Epico 2. Cobre FR2 e a parte visual da biblioteca de componentes (Componente 8 da arquitetura). Sem React/bundler (ADR-4) — Web Components ou classes JS puras.

## Descricao
Como no-coder, quero ver uma palette lateral esquerda com 8 componentes v1 e poder iniciar drag-and-drop a partir dela, para montar paginas.

## Tarefas tecnicas
1. [ ] Criar `public/builder/palette.js` com definicao de cada componente v1: `{ type, label, icon, defaultProps }` para Container, Heading, Text, Button, Input, Form, List, Table, Image.
2. [ ] Renderizar palette em `public/builder/index.html` (coluna esquerda do shell): cards arrastaveis com `draggable="true"` e `dataTransfer.setData("application/x-builder-component", JSON.stringify(componentDef))`.
3. [ ] Adicionar `public/builder/vendor/interactjs.min.js` (vendorada) para gestos finos (opcional v1; HTML5 nativo basta).
4. [ ] Estilizar palette em `public/builder/style.css` seguindo principios UI/UX da §3 do PRD (zero terminologia de dev — labels: "Texto", "Botao", "Lista").
5. [ ] Tooltips em hover descrevendo o que cada componente faz (1 frase, sem jargao).

## Arquivos envolvidos
- `public/builder/palette.js` - criar (definicoes + render)
- `public/builder/index.html` - modificar (coluna palette)
- `public/builder/style.css` - modificar (estilos)
- `public/builder/vendor/interactjs.min.js` - criar (vendor)

## Definition of Done
- [ ] Palette mostra 8 componentes com labels em portugues sem jargao
- [ ] Cada item e arrastavel (cursor muda, dataTransfer setado)
- [ ] Tooltips aparecem em hover, descrevem em 1 frase
- [ ] Layout responsivo: coluna fixa 240px na esquerda
- [ ] Sem erros no console
- [ ] F5 preserva estado
