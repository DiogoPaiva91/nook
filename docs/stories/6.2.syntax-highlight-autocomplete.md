# Story 6.2: Autocomplete e go-to-definition via TS LSP

## Contexto
Epico 6. Cobre FR17 (autocomplete) + AC do epico ("ir-para-definicao funciona"). Monaco tem suporte nativo a LSP via web worker; precisamos configurar `tsconfig.json` do projeto e abrir conexao LSP.

## Descricao
Como Diogo, quero autocomplete TypeScript real e go-to-definition dentro do editor Monaco do builder, para navegar como em Cursor/VSCode.

## Tarefas tecnicas
1. [ ] Configurar Monaco TypeScript worker apontando para o `tsconfig.json` do projeto ativo. Usar `monaco.languages.typescript.typescriptDefaults.setCompilerOptions(...)` carregando do projeto.
2. [ ] Carregar tipos do projeto (node_modules + arquivos do projeto) via `addExtraLib`. Endpoint `GET /api/builder/projects/:id/typedefs` que retorna tipos relevantes (limitar a `@types/*` + `*.d.ts` do projeto + arquivos do `app/`, `lib/`, `db/`).
3. [ ] Implementar go-to-definition: shortcut F12 ou Ctrl+click; resolve via Monaco (que ja sabe quando o LSP esta ativo).
4. [ ] Hover-info funcionando (tooltip de tipos). Erros em vermelho na gutter.
5. [ ] Cache local dos typedefs com invalidation por watcher (Story 2.10) quando arquivos relevantes mudarem.

## Arquivos envolvidos

- `lib/builder/router.js` - modificar (rota typedefs)
- `lib/builder/indexer.js` - modificar (helper para coletar typedefs)
