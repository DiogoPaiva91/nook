# Story 1.1: Router do Builder + Metadata SQLite

## Contexto
Primeira pedra do Epico 1 (Fundacao). Cobre Componente 1 (Router) e Componente 2 (Metadata) da arquitetura. Sem isso, nenhuma rota `/api/builder/*` existe e nao ha onde registrar projetos. O router e o ponto de entrada delegado pelo `server.js` antes da cadeia de `if`s atual; a metadata mora no mesmo `data/jarvis.db` (ADR-6).

## Descricao
Como Diogo (dev do Hub), quero um router dedicado em `lib/builder/` e tabelas SQLite `builder_projects` e `builder_preview_processes` para que toda funcionalidade subsequente do Modo de Codigo tenha pontos de extensao isolados, sem tocar no monolito do `server.js`.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/router.js` exportando `handle(req, res): boolean` com matcher para `/api/builder/*` e `/builder/*` (placeholder ainda sem rotas, retorna 404 JSON dentro do prefixo).
2. [ ] Criar `lib/builder/db.js` com migrations idempotentes para `builder_projects (id TEXT PK, name, path, stack, supabase_url, created_at, updated_at)` e `builder_preview_processes (project_id, pid, port, started_at)`. Recebe handle do db por injecao (default = `require("../db")`).
3. [ ] Modificar `server.js`: adicionar 3 linhas no topo do callback do `createServer` chamando `require("./lib/builder/router").handle(req, res)`; se `true`, `return`.
4. [ ] Garantir migrations rodam no boot (chamar `lib/builder/db.js init()` em algum bootstrap existente, p.ex. apos abrir o db principal).
5. [ ] Smoke test manual: `curl /api/builder/health` retorna 200 JSON `{ ok: true }`.

## Arquivos envolvidos
- `lib/builder/router.js` - criar (handler com matcher de prefixo + rota `/api/builder/health`)
- `lib/builder/db.js` - criar (migrations idempotentes)
- `server.js` - modificar (3 linhas de delegacao)
- `data/jarvis.db` - efeito colateral (novas tabelas)

## Definition of Done
- [ ] `curl http://localhost:<porta>/api/builder/health` responde 200 com JSON
- [ ] `SELECT name FROM sqlite_master WHERE type='table' AND name LIKE 'builder_%'` retorna 2 tabelas
- [ ] Reiniciar o Hub nao duplica colunas nem quebra (migration idempotente)
- [ ] Rotas existentes do Jarvis Hub (chat, workers, hub) continuam funcionando identicas
- [ ] Sem erros no console
- [ ] F5 preserva estado
