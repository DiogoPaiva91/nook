# Story 1.3: Template-base Next.js + Supabase versionado

## Contexto
Story do Epico 1. Cobre Componente 3 da arquitetura. ADR-7: cópia interna versionada (não `npx create-next-app`) para garantir determinismo de versões e funcionar offline (NFR8). Toda criacao de projeto via 1.2 usa este template.

## Descricao
Como mantenedor do Hub, quero um template-base completo de Next.js 15 + TS + Tailwind + `@supabase/ssr` + Drizzle versionado em `templates/nextjs-supabase/` para que cada projeto novo nasça idiomatico, com auth funcional, lockfile pinado e `.mcp.json` ja configurado.

## Tarefas tecnicas
1. [ ] Criar `templates/nextjs-supabase/` com scaffold Next.js 15 App Router + TypeScript + Tailwind: `app/layout.tsx`, `app/page.tsx`, `app/globals.css`, `tsconfig.json`, `next.config.mjs`, `tailwind.config.ts`, `postcss.config.js`, `package.json` (com versions pinadas) e `package-lock.json`.
2. [ ] Adicionar `@supabase/ssr` + middleware de auth em `middleware.ts` + utils em `lib/supabase/{client,server,middleware}.ts`.
3. [ ] Adicionar Drizzle: `drizzle.config.ts`, `db/index.ts`, `db/schema.ts` (vazio) + `supabase/config.toml` + pasta `supabase/migrations/` vazia.
4. [ ] Criar `templates/nextjs-supabase/.env.example` (URL/anon key/service role placeholders) e `templates/nextjs-supabase/.mcp.json` apontando para o Supabase MCP server com placeholders de credenciais.
5. [ ] Criar `templates/nextjs-supabase/builder.json` com schema vazio (version:1, project, entities:[], pages:[], files:{}) e `README.md` com instrucoes de `supabase start` local OU colar URL/key da Supabase Cloud (FR10d).
