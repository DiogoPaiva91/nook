# Story 5.1: Representacao de workflow no `builder.json`

## Contexto
Epico 5 (Workflows visuais). Cobre FR15 do PRD parcialmente. Conforme §9 da arquitetura: workflows entram como novo no top-level (`workflows[]`) do `builder.json`, seguindo o mesmo padrao de paginas/entidades.

## Descricao
Como mantenedor do builder, quero estender o schema de `builder.json` com `workflows[]` (nodes, edges, triggers, steps) validados, para que codegen e UI tenham um contrato claro.

## Tarefas tecnicas
1. [ ] Estender `lib/builder/schema.js` (Story 2.1) com definicao de `workflow`: `{ id, name, trigger: {...}, nodes: [{ id, type, props, next }], edges: [...] }`. Tipos de trigger: `http`, `db.insert/update/delete` (Database Webhook), `auth.user_created`, `schedule.cron`.
2. [ ] Tipos de step v1: `http_request`, `db_query`, `db_insert`, `db_update`, `db_delete`, `email_send`, `storage_upload`, `edge_function_call`, `webhook`, `condition`, `loop`, `delay`.
3. [ ] Validacao incluir: cada node tem id unico; edges referenciam ids existentes; trigger obrigatorio.
4. [ ] Atualizar `docs/architecture.md` com exemplo de workflow no `builder.json`.
5. [ ] Endpoint `POST /api/builder/projects/:id/schema/patch` ja existente aceita patches em `workflows[]`.

## Arquivos envolvidos
- `lib/builder/schema.js` - modificar (workflows)
- `docs/architecture.md` - modificar (exemplo §6)
- `lib/builder/store.js` - referenciado (validacao)

## Definition of Done
- [ ] Patch adicionando workflow valido aceita
- [ ] Patch com edge para node inexistente rejeita
- [ ] Trigger obrigatorio: workflow sem trigger rejeita
- [ ] Doc atualizada com exemplo legivel
- [ ] Validacao roda em <50ms para workflows com 50 nodes
- [ ] Sem erros no console
- [ ] F5 preserva estado

Complexidade: baixa
