# Story 7.1: Extrair core do builder + runtime para repo GitHub publico

## Contexto
Epico 7 (Open-source). Cobre FR20 do PRD. Conforme §5 da arquitetura ("Regra de ouro"): `lib/builder/`, `templates/`, `public/builder/` formam o core publico, sem importar nada do resto do Jarvis Hub privado.

## Descricao
Como mantenedor, quero criar um repositorio GitHub publico contendo apenas o core do builder + runtime do template, mantendo o Jarvis Hub privado em outro repo, para liberar a base open-source sem expor dados pessoais.

## Tarefas tecnicas
1. [ ] Auditoria: rodar `grep -r "require.*lib/(chat|workers|hub|providers/)" lib/builder/ templates/ public/builder/` — listar imports proibidos. Refatorar quaisquer dependencias indevidas (ex.: `lib/db.js` -> abstrair via injecao na story 1.1, ja preparado).
2. [ ] Criar repo GitHub `jarvis-builder` (ou nome a definir): copia de `lib/builder/` + `templates/` + `public/builder/` + raiz com `package.json`, `server.js` minimo (Node http) que so monta o router do builder.
3. [ ] No Hub privado, transformar `lib/builder/` em git submodule ou simples reference por copia versionada (decidir: submodule vs copia manual sincronizada). Documentar processo.
4. [ ] Setup `package.json` do repo publico: deps minimas (`prettier`, `archiver`, `proper-lockfile`, `ajv`, `simple-git`, `better-sqlite3`); script `npm start` que sobe o builder em porta 4000.
5. [ ] CI pipeline ja em 7.3; aqui apenas garantir o repo standalone clonavel + `npm install && npm start` funciona.

## Arquivos envolvidos
- Repo novo `jarvis-builder/` - criar (copia + entrypoint)
- `lib/builder/db.js` - referenciado (injecao ja preparada)
- `package.json` no repo novo - criar
- `server.js` no repo novo - criar (entrypoint minimo)

## Definition of Done
- [ ] Repo publico criado, clone limpo passa `npm install && npm start`
- [ ] `curl http://localhost:4000/builder/` carrega UI do builder
- [ ] Auditoria de imports nao detecta dependencias do Hub privado
- [ ] Hub privado continua funcionando apos refatoracao (testes manuais)
- [ ] Estrategia de sync (submodule vs copia) documentada em CONTRIBUTING.md
- [ ] Sem erros no console
- [ ] F5 preserva estado
