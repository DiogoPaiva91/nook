# Story 2.6: Modelo de dados visual + codegen Drizzle + migrations Supabase + RLS

## Contexto
Epico 2. Cobre FR7 (modelo de dados, schema Postgres, RLS padrao), ADR-1 (Drizzle so para types/queries), item 4 da §10 (migrations sao SQL puras do Supabase CLI). Aba lateral "Modelo de dados" no canvas.

## Descricao
Como no-coder, quero criar entidades, campos e relacoes visualmente e ver migration SQL + types Drizzle + policies RLS gerados automaticamente, sem tocar em SQL.

## Tarefas tecnicas
1. [ ] Criar `public/builder/data-model.js` com UI: aba lateral (ou modal) listando entidades; botao "Nova entidade"; form de campo (name, type, required, primary, relation). Patches em `entities[]` do `builder.json`.
2. [ ] Criar `lib/builder/codegen/schema.js`: a partir de `entities[]`, gerar `db/schema.ts` (Drizzle), `supabase/migrations/<timestamp>_<entity>.sql` (CREATE TABLE) e `supabase/migrations/<timestamp>_<entity>_rls.sql` (RLS policies por modo `public | authenticated | owner`).
3. [ ] Detectar diff entre estado anterior e novo: gerar **nova migration** incremental (ALTER TABLE ADD/DROP COLUMN), nunca editar migrations antigas.
4. [ ] Suporte a tipos basicos: uuid, text, integer, boolean, timestamp, jsonb, references (FK).
5. [ ] Botao "Aplicar ao banco" na UI dispara `supabase db push` (via spawn) no contexto do projeto; mostrar log no painel.
