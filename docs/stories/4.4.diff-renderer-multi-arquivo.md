# Story 4.4: Diff renderer multi-arquivo com aceitar/rejeitar por arquivo

## Contexto
Epico 4. Cobre FR13 (aceitar, rejeitar, editar diff). Componente 7 da arquitetura. Critico para confianca do usuario no agente — sem diff-review nao tem agente.

## Descricao
Como Diogo, quero ver os diffs por arquivo gerados pelo agente antes de aplicar, com opcoes aceitar/rejeitar/editar por arquivo, para nunca ter mudanca surpresa no codigo.

## Tarefas tecnicas
1. [ ] Modificar `lib/builder/agent.js`: write_file do agente NAO escreve direto — registra em uma "staging area" (memoria + tabela `builder_pending_changes`).
2. [ ] Criar `lib/builder/diff.js` com `computeDiff(oldContent, newContent)` em formato unified diff; suportar criar/modificar/deletar.
3. [ ] Endpoint `GET /api/builder/projects/:id/agent/pending` lista mudancas pendentes; `POST .../pending/:fileId/{accept,reject}` aplica ou descarta uma; `POST .../pending/accept-all` aplica todas.
4. [ ] UI: ao final do streaming, painel chat exibe lista de arquivos com diff colorido (verde/vermelho), botoes por arquivo + bulk accept/reject.
5. [ ] Aceitar dispara write real (passa pelo watcher Story 2.10); rejeitar limpa staging sem tocar disco.

## Arquivos envolvidos
- `lib/builder/agent.js` - modificar (staging em vez de write direto)
- `lib/builder/diff.js` - criar
- `lib/builder/db.js` - modificar (tabela pending_changes)
- `lib/builder/router.js` - modificar (rotas pending)
- `public/builder/chat.js` - modificar (UI de diff)
- `public/builder/style.css` - modificar (cores diff)

## Definition of Done
- [ ] Mensagem "adicione botao de deletar em cada linha" gera staging com 1-3 arquivos
- [ ] Diffs aparecem coloridos (verde adicionado / vermelho removido)
- [ ] Aceitar 1 arquivo aplica so ele; outros continuam pendentes
- [ ] Rejeitar nao deixa estado sujo (filesystem inalterado, staging limpa)
- [ ] Aceitar todos aplica em ordem e atualiza `builder.json` quando aplicavel
- [ ] Sem erros no console
- [ ] F5 preserva pendencias (banco persiste)
