# Story 5.3: Codegen de workflows -> route handlers / server actions / Edge Functions

## Contexto
Epico 5. Cobre FR15 + FR16a do PRD. Conforme §6 do PRD: triggers de DB e jobs agendados -> Edge Functions; fluxos sincronos ligados a UI -> server actions; HTTP/webhooks -> route handlers.

## Descricao
Como no-coder, quero que cada workflow gere automaticamente o codigo certo (route handler, server action OU Edge Function Deno) conforme o tipo de trigger, para nao escolher tipo de codigo manualmente.

## Tarefas tecnicas
1. [ ] Criar `lib/builder/codegen/workflow.js` que recebe um `workflow` e decide o destino: trigger HTTP -> `app/api/<slug>/route.ts`; trigger DB/auth -> `supabase/functions/<slug>/index.ts`; trigger UI/sincrono -> `app/actions/<slug>.ts` (server action).
2. [ ] Para cada step type, ter um emissor de codigo idiomatico (HTTP -> `fetch`, DB -> Drizzle query, email -> Supabase `auth.api.sendEmail` ou Resend, etc.).
3. [ ] Topologia: ordenar nodes por arestas (DAG simples), emitir como sequencia de awaits; condicionais viram `if`, loops viram `for-of`.
4. [ ] Triggers Supabase (FR16a): para `db.insert/update/delete`, gerar tambem o registro de Database Webhook em `supabase/migrations/<ts>_webhook_<slug>.sql` apontando para a Edge Function gerada.
5. [ ] Header `// @builder:managed` igual ao codegen de paginas; integrado no orquestrador `lib/builder/codegen/index.js`.

## Arquivos envolvidos
- `lib/builder/codegen/workflow.js` - criar
- `lib/builder/codegen/index.js` - modificar (incluir workflow)
- `lib/builder/codegen/schema.js` - referenciado (Database Webhook na migration)

## Definition of Done
- [ ] Workflow "ao criar Agent, enviar webhook para URL X" gera Edge Function + migration de webhook
- [ ] Workflow "GET /api/health responde OK" gera `app/api/health/route.ts` valido
- [ ] Server action gerada chamavel de um Form (binding via Story 2.7)
- [ ] `npm run build` no projeto passa apos gerar 3 workflows distintos
- [ ] AC do epico: "ao criar agent, enviar webhook" funciona em preview
- [ ] Sem erros no console
- [ ] F5 preserva estado
