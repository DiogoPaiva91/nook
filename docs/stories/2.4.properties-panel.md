# Story 2.4: Properties panel dinamico por tipo de componente

## Contexto
Epico 2. Cobre FR3 do PRD. Coluna direita do builder. Reage a `builder:selection` (Story 2.3) e edita props via patches em `builder.json`.

## Descricao
Como no-coder, quero ver um painel de propriedades editavel ao selecionar um componente, com campos especificos do tipo (texto, cor, tamanho, evento onClick, binding com dado), para customizar visualmente sem escrever codigo.

## Tarefas tecnicas
1. [ ] Criar `public/builder/properties.js` com definicao declarativa por tipo: `{ Button: [{ name:"label", type:"text" }, { name:"variant", type:"select", options:[...] }, { name:"onClick", type:"action" }], ... }`.
2. [ ] Renderizar form dinamico no painel direito a partir do tipo do no selecionado; preencher valores atuais do `builder.json`.
3. [ ] Onchange de cada campo gera JSON Patch `{ op:"replace", path:".../props/<campo>", value }` e envia para `schema/patch`. Debounce 200ms para inputs de texto.
4. [ ] Suportar tipos de campo: text, number, select, color, boolean, action (lista de acoes pre-definidas), entityRef (preenchido pela story 2.7), bindingRef.
5. [ ] Vazio quando nada selecionado: mensagem "Selecione um componente para editar suas propriedades".

## Arquivos envolvidos
- `public/builder/properties.js` - criar (definicoes + render + patch)
- `public/builder/index.html` - modificar (coluna direita)
- `public/builder/style.css` - modificar (estilos do painel)
- `public/builder/app.js` - modificar (escutar `builder:selection`)

## Definition of Done
- [ ] Selecionar Botao mostra campos: label, variant, onClick
- [ ] Editar label re-renderiza canvas em <= 200ms (NFR1)
- [ ] Patch correspondente persiste em `builder.json`
- [ ] Trocar selecao reseta o painel para o novo tipo
- [ ] Sem componente selecionado mostra placeholder
- [ ] Sem erros no console
- [ ] F5 preserva estado
