# Story 1.4 — Limpar lista

**Épico:** 1 — CRUD da Lista
**Status:** Done
**Owner:** Dev
**Estimativa:** S (≤ 0.25d)

## Story
Como **usuário que terminou as compras**, quero **limpar toda a lista de uma vez**, para que eu possa **começar uma nova lista sem ter que remover item por item**.

## Acceptance Criteria
1. Existe um botão **"Limpar lista"** no rodapé (`#clear-all`), visível **apenas quando há itens**.
2. Ao clicar, exibe `confirm()` com mensagem clara ("Apagar todos os itens da lista?").
3. Confirmando: lista zera, estado vazio aparece (Story 3.1), botão "Limpar lista" desaparece.
4. Cancelando: nada muda.
5. localStorage é gravado com `{ items: [] }`.

## Dev Notes
**Origem:** PRD §6 / Story 1.4, FR4. Architecture.md §10 ADR-007 (confirmar só aqui — ação não-trivial de reverter).

**Contrato:**
- `store.clearAll(): void` — `state.items = []` + persiste.

**Visibilidade do botão:** `render.js` esconde/mostra `#clear-all` no mesmo branch do estado vazio (mostra quando `total > 0`, esconde quando `total === 0`). Ver Story 3.1.

**Por que `confirm()` nativo (e não modal custom):** zero dependências, acessível por padrão, suficiente pro caso. NFR1 (sem build) reforça.

**Edge case:** usuário pode estar em modo privado / `localStorage` desabilitado. `clearAll` ainda funciona em memória; `save()` engole erro silenciosamente (architecture §9).
