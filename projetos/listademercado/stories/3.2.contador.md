# Story 3.2 — Contador

**Épico:** 3 — Feedback Visual
**Status:** Done
**Owner:** Dev
**Estimativa:** XS (≤ 0.1d)

## Story
Como **usuário no mercado**, quero **ver de relance quantos itens já comprei e quantos faltam**, para que eu possa **saber se já está na hora de ir pro caixa**.

## Acceptance Criteria
1. Header exibe contador no formato **`X de Y itens`**, onde `X` = comprados e `Y` = total.
2. Atualiza **em tempo real** após qualquer mutação: add, toggle, remove, clear.
3. Singular/plural pode ser ignorado na v1 ("1 de 1 itens" é aceitável — escopo enxuto).
4. Quando `total === 0`, contador exibe "0 de 0 itens" ou é ocultado (qualquer dos dois é aceitável; ver `mock.html` pra direção visual).

## Dev Notes
**Origem:** PRD §6 / Story 3.2, FR6. Architecture.md §5.4.

**Implementação:** `render.js` calcula no início do `renderList`:
```js
const total = state.items.length;
const bought = state.items.filter(i => i.bought).length;
counterEl.textContent = `${bought} de ${total} itens`;
```

**Por que `textContent` e não template literal injetado em `innerHTML`:** segurança por construção (architecture §12). Mesmo com números, padrão é `textContent`.

**Reatividade:** o contador é atualizado pelo mesmo `refresh()` que redesenha a lista — não precisa de subscriber separado (architecture §8 — "render-everything-on-change").

**Bug encontrado:**
- `render.js:47` calculava `pendentes = state.items.filter((i) => !i.bought).length` e renderizava `${pendentes} de ${total} itens` — **invertido**.
- AC1 da story (e PRD §6 / FR6) define **X = comprados**.
- `mock.html:13` mostra `2 de 5 itens` com 2 itens em estado `.bought` — confirmando que **2 = comprados**.
- Comportamento incorreto antes do fix: marcar todos os itens mostraria `0 de N`; lista nova mostraria `N de N`.

**Fix aplicado** (`src/render.js`):
```diff
- const pendentes = state.items.filter((i) => !i.bought).length;
+ const comprados = state.items.filter((i) => i.bought).length;
...
- counterEl.textContent = `${pendentes} de ${total} itens`;
+ counterEl.textContent = `${comprados} de ${total} itens`;
```

**Validação manual pós-fix:**
- AC1 ✅ formato `${comprados} de ${total} itens` agora alinha com PRD/mock.
- AC2 ✅ contador chamado em todo `refresh()` — reativo.
- AC3 ✅ singular/plural ignorado (escopo v1).
- AC4 ✅ `render.js:53-54` esconde counter quando `total === 0` (escolha: oculto, não "0 de 0").

**Recomendação ao QA:** validar tempo real após cada operação (add → "0 de 1", toggle → "1 de 1", remove → "0 de 0"/oculto, clear → oculto) — caso 2 e 3 são os que falhavam antes.
