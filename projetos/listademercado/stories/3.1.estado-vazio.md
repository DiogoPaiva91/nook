# Story 3.1 — Estado vazio

**Épico:** 3 — Feedback Visual
**Status:** Done
**Owner:** Dev
**Estimativa:** XS (≤ 0.1d)

## Story
Como **usuário que abre o app pela primeira vez (ou após limpar a lista)**, quero **uma mensagem amigável indicando o que fazer**, para que eu **não fique olhando uma tela em branco sem saber por onde começar**.

## Acceptance Criteria
1. Quando `state.items.length === 0`, a UL da lista é **escondida** e o elemento `#empty-state` é **mostrado**.
2. Mensagem exibida orienta o usuário (ex.: "Sua lista está vazia" ou similar — texto definido no `index.html`).
3. Botão **"Limpar lista"** fica oculto neste estado (não há nada para limpar).
4. Contador "X de Y itens" pode ser ocultado ou exibido como "0 de 0 itens" (ver Story 3.2).
5. Form de adicionar **continua visível e focado** — é o caminho de saída do estado vazio.

## Dev Notes
**Origem:** PRD §6 / Story 3.1, FR7. Architecture.md §5.4 (render.js controla visibilidade).

**Implementação:** `render.js` calcula `total = state.items.length` e:
- `total === 0`: `list.hidden = true`, `clearBtn.hidden = true`, `emptyState.hidden = false`
- `total > 0`: inverso

Atributo `hidden` (HTML5) é preferido sobre `style.display` — semântico, sem necessidade de CSS extra (browser aplica `display: none` por padrão).

**Por que esconder o "Limpar lista":** botão para uma ação sobre 0 itens é ruído — o usuário pode clicar achando que faz alguma coisa. Esconder é a UX correta.
