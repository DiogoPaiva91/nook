# Story 1.2 — Marcar como comprado

**Épico:** 1 — CRUD da Lista
**Status:** Done
**Owner:** Dev
**Estimativa:** S (≤ 0.25d)

## Story
Como **usuário no mercado**, quero **marcar um item como comprado** ao colocá-lo no carrinho, para que eu possa **ver de relance o que ainda falta**.

## Acceptance Criteria
1. Cada item da lista tem um **checkbox à esquerda**.
2. Clicar no checkbox alterna `bought` (true/false) no estado.
3. Item marcado recebe classe visual `bought` (texto riscado, cor mais clara — ver `mock.html`).
4. Contador `X de Y itens` (Story 3.2) atualiza imediatamente após o toggle.
5. Estado persiste — recarregar a página mantém os marcados (Story 2.1).
6. Itens comprados podem ser ordenados depois dos pendentes na renderização (`render.js`).

## Dev Notes
**Origem:** PRD §6 / Story 1.2, FR2. Architecture.md §5.2 (`store.toggleBought`), §5.4 (render: ordenação pendentes/comprados), §8 (re-render completo).

**Contrato envolvido:**
- `store.toggleBought(id: string): void` — busca por id, inverte `bought`, persiste. No-op se id não existe.

**Implementação atual:**
- Listener é **delegado** no `<ul>` (architecture §10 ADR-006), não em cada checkbox individual — sobrevive ao re-render full.
- Checkbox no DOM tem `data-action="toggle"` e `data-id="<uuid>"`.
- Listener filtra por `target.dataset.action === "toggle"` e chama `toggleBought(target.dataset.id)`.

**Justificativa do delegate:** lista é redesenhada inteira a cada mutação (ADR-004); listeners por item exigiriam reanexar a cada render — desperdício.

## Tasks / Subtasks
- [x] `render.js`: cada `<li>` recebe `<input type="checkbox" data-action="toggle" data-id="...">` + classe `bought` quando aplicável
- [x] `main.js`: listener `change` no `#list` filtrando por `data-action="toggle"`
- [x] `store.js`: `toggleBought(id)` localiza item, inverte `bought`, persiste
- [x] `styles/main.css`: regra `.item.bought .item-name` com `text-decoration: line-through` e cor secundária
