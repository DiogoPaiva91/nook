# Story 1.3 — Remover item

**Épico:** 1 — CRUD da Lista
**Status:** Done
**Owner:** Dev
**Estimativa:** S (≤ 0.25d)

## Story
Como **usuário**, quero **remover um item específico da lista**, para que eu possa **corrigir um item digitado errado ou retirar algo que não quero mais comprar**.

## Acceptance Criteria
1. Cada item tem um botão `×` à **direita**, com `aria-label` descritivo.
2. Clicar no `×` remove o item da lista **sem confirmação** (decisão consciente — ver ADR-007).
3. Item desaparece do DOM imediatamente.
4. Contador (`X de Y itens`) atualiza.
5. Persistência grava o novo estado (Story 2.1).
6. Se era o último item, app entra no estado vazio (Story 3.1).

## Dev Notes
**Origem:** PRD §6 / Story 1.3, FR3. Architecture.md §10 ADR-007 (sem confirmação aqui; confirma só no clearAll), §5.2 (`store.removeItem`), §5.4 (render).

**Contrato:**
- `store.removeItem(id: string): void` — `state.items = state.items.filter(i => i.id !== id)` + persiste.

**Por que sem confirmação:** remover 1 item é trivialmente reversível (re-digitar). `confirm()` adicionaria atrito num gesto rotineiro. A ação destrutiva real é `clearAll` (Story 1.4).

**Acessibilidade:** botão `×` é texto simbólico — exige `aria-label="Remover <nome do item>"` para leitores de tela.

**Event delegation:** listener `click` está no `<ul>`, usa `target.closest('[data-action="remove"]')` para localizar o botão (o usuário pode acertar o `<svg>` ou span interno — `closest` resolve).
