# Story 1.1 — Adicionar item

**Épico:** 1 — CRUD da Lista
**Status:** Done
**Owner:** Dev
**Estimativa:** S (≤ 0.5d)

## Story
Como **usuário no mercado**, quero **digitar o nome de um item e adicioná-lo à minha lista** rapidamente, para que eu possa **anotar o que preciso comprar sem fricção**.

## Acceptance Criteria
1. Existe um campo de texto e um botão "Adicionar" no topo da tela (form `#add-form`, input `#item-input`).
2. Submeter via **Enter** no input ou **clique** no botão Adicionar tem o mesmo efeito (handler único de `submit`).
3. Texto vazio ou apenas whitespace **não adiciona** item (validação por `trim()` no `store.addItem`).
4. Input limita a **80 caracteres** (`maxlength` no HTML).
5. Após adicionar com sucesso: input é limpo, foco volta ao input, lista re-renderiza com o item novo.
6. Item novo aparece **no topo** da lista (`unshift`).

## Dev Notes
**Origem:** PRD §6 / Story 1.1, FR1, FR8. Architecture.md §5.1 (main), §5.2 (store), §6 (modelo Item), §7 (fluxo "Adicionar item").

**Contrato envolvido:**
- `store.addItem(name: string): boolean` — retorna `false` quando vazio (sem mutação).
- Item criado: `{ id: crypto.randomUUID(), name: trim, bought: false, createdAt: Date.now() }`.

**Fluxo de dados:**
```
submit → e.preventDefault() → addItem(input.value)
  → trim → unshift → persist() → return true
  → input.value = "" + refresh() + input.focus()
```

**Regras importantes:**
- Item duplicado é permitido (FR8 — "2 leites" é caso real).
- Persistência é write-through: cada `addItem` grava no `localStorage` antes de retornar (ver Story 2.1).
- Render usa `textContent`, nunca `innerHTML` — não há risco de XSS via texto do item (architecture §12).
